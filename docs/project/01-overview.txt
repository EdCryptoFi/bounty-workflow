1. O QUE É
-------------------------------------------------------------------------------
SaaS em PT-BR para "bounty hunters" / caçadores de recompensas em protocolos
cripto. O usuário cadastra campanhas (uma por protocolo/bounty), quebra em
tarefas (steps), anexa arquivos de evidência, define prazos e recebe
lembretes automáticos por e-mail. Plano freemium com trial de 14 dias +
assinatura mensal via Mercado Pago (BR) ou Stripe (internacional).

Público-alvo: usuários brasileiros que executam tarefas pagas em protocolos
DeFi/Web3 (airdrops, quests, bounties) e precisam organizar prazos, evidências
e pagamentos em um só lugar.

-------------------------------------------------------------------------------
2. PROPOSTA DE VALOR
-------------------------------------------------------------------------------
- Timeline única com todas as campanhas ordenadas por deadline
- Quebra em steps com status (todo / in_progress / done)
- Workflow canvas visual (react-flow) para planejamento
- Upload de anexos por campanha (bucket privado Supabase, RLS)
- Lembretes por e-mail via cron diário (Vercel + Resend)
- Export CSV com proteção contra CSV injection (A4)
- Catálogo público de protocolos suportados
- Área admin para cadastrar/editar protocolos

5. ARQUITETURA EM ALTO NÍVEL
-------------------------------------------------------------------------------
Next.js App Router com três route groups:

    src/app/(public)   — landing, catálogo de protocolos, páginas legais
    src/app/(auth)     — login, signup, callback OAuth, signout
    src/app/(app)      — dashboard, campanhas, admin, reminders, settings

Middleware (src/middleware.ts) refresca sessão Supabase em todas as requests
exceto assets estáticos, /api/cron e /api/webhooks (webhooks autenticam via
HMAC próprio).

Dados via Server Components + Server Actions. RLS ativo em TODAS as tabelas
(FORCE RLS). Service role key só usada em createAdminClient() para cron e
webhooks.

Fluxo típico de mutação:
    Form (client) -> Server Action -> Supabase (RLS) -> revalidatePath

Fluxo de pagamento:
    Usuário clica Upgrade -> Server Action cria Preapproval MP
    -> redireciona pro checkout -> MP chama /api/webhooks/mercadopago
    -> valida HMAC + idempotência (webhook_events) -> atualiza subscriptions

Fluxo de cron:
    Vercel cron (09:00 UTC) -> GET /api/cron/reminders (CRON_SECRET)
    -> busca reminders pending com trigger_at <= now -> envia e-mail (Resend)
    -> atualiza status + loga em notification_logs
