MÓDULO: SUPABASE CLIENT
-------------------------------------------------------------------------------
Pasta:   src/lib/supabase/
Papel:   Fábricas de cliente Supabase para 3 contextos distintos.

 client.ts        createClient() para Client Components ("use client").
                  Usa createBrowserClient de @supabase/ssr.

 server.ts        createClient() para Server Components/Actions.
                  Usa createServerClient lendo cookies() de next/headers.
                  Também expõe createAdminClient() com SUPABASE_SERVICE_ROLE_KEY
                  — bypass de RLS. USAR SÓ em cron e webhooks.

 middleware.ts    updateSession(request) — refresca o token e sincroniza
                  cookies no NextResponse. Chamado pelo middleware edge.

MÓDULO: CAMPAIGNS (queries + actions)
-------------------------------------------------------------------------------
Pasta: src/lib/campaigns/

 queries.ts
   listCampaignsForTimeline()
     SELECT campanhas do user + join implícito (contagem de steps via RPC
     ou subquery) + cálculo de days_to_deadline. Ordenado por deadline.
     Usado por /dashboard, /campaigns, /archive.

   getCampaignById(id)
     Fetch da campanha com steps (ordered by order_index) e attachments.
     Usado por /campaigns/[id].

 actions.ts  (use server)
   createCampaignAction(formData)
     Valida com Zod (title, description, protocol_id, deadline,
     estimated_value_brl). Insere em campaigns (user_id via RLS).
     revalidatePath("/dashboard") + redirect(`/campaigns/${id}`).

   addStepAction(campaignId, formData)
     Calcula next order_index (MAX+1 por campanha). Insere step.
     revalidatePath(`/campaigns/${campaignId}`).

   toggleStepAction(stepId, nextStatus)
     Atualiza step.status; se "done" seta completed_at = now.

   archiveAction(id) / unarchiveAction(id)
     Seta archived_at / limpa; revalida /dashboard e /archive.

MÓDULO: PAYMENTS
-------------------------------------------------------------------------------
Pasta: src/lib/payments/

 config.ts
   getActiveProvider(): "mercadopago" | "stripe" | null
   - null se paymentsEnabled=false
   - "mercadopago" se MERCADOPAGO_ACCESS_TOKEN está setado (preferência BR)
   - "stripe" se STRIPE_SECRET_KEY + STRIPE_WEBHOOK_SECRET
   - null caso contrário

 mercadopago.ts
   createPreapproval({ userId, email })
     POST /preapproval — subscription recorrente. external_reference = userId
     (NUNCA email). Retorna init_point (URL de checkout).
   getPreapproval(id)
     Fallback quando webhook chega sem secret — valida evento via API.
   verifyWebhookSignature(headers, rawBody)
     Valida x-signature (HMAC-SHA256 com MERCADOPAGO_WEBHOOK_SECRET).

 stripe.ts
   Cliente Stripe via fetch puro (sem SDK pra manter bundle pequeno).
   Cria Checkout Sessions e valida assinatura com crypto.subtle.verify.

 idempotency.ts
   claimEvent(provider, eventId, type, payload)
     INSERT em webhook_events com UNIQUE(provider, provider_event_id).
     Se já existe -> retorna "duplicate" e handler retorna 200 sem reprocessar.
   finalizeEvent(id, status, error?)
     UPDATE com processed_at e error_message.

 sendEmail({ to, subject, html, text? })
   POST https://api.resend.com/emails com Authorization: Bearer RESEND_API_KEY.
   from = RESEND_FROM. Retorna { ok: true, id } ou { ok: false, error }.

API ROUTES
-------------------------------------------------------------------------------

 src/app/api/health/route.ts
   GET /api/health -> 200 { ok: true, ts }
   Smoke test básico. Usado pelos scripts de verificação de deploy.

 src/app/api/cron/reminders/route.ts
   GET /api/cron/reminders (Vercel cron — vercel.json — 0 9 * * *)
   Auth: header "x-cron-secret" == CRON_SECRET.
   Fluxo:
     1. createAdminClient() (bypass RLS pra varrer todos os users).
     2. SELECT reminders WHERE status='pending' AND trigger_at <= now.
     3. Pra cada reminder: envia e-mail via Resend (canais push/in_app skip).
     4. UPDATE reminders status='sent' | 'failed'; INSERT notification_logs.

 src/app/api/export/campaigns/route.ts
   GET /api/export/campaigns?scope=active|archived|all
   Auth: session normal (RLS enforça escopo do user).
   Busca campanhas + protocol name, passa por csvEscape(), devolve
   text/csv com Content-Disposition: attachment;
   filename="bountyworkflow-{scope}-{YYYY-MM-DD}.csv".

 src/app/api/webhooks/stripe/route.ts
   POST webhook Stripe. Lê rawBody, valida Stripe-Signature com
   crypto.subtle. claimEvent -> processa (invoice.paid, subscription.*) ->
   finalizeEvent. Atualiza subscriptions.

 src/app/api/webhooks/mercadopago/route.ts
   POST webhook MP. Valida x-signature via HMAC (se secret setado) OU
   faz fallback getPreapproval(id) pra validar via API. claimEvent ->
   processa -> finalizeEvent. Atualiza subscriptions.

 src/app/api/webhooks/asaas/route.ts
   410 Gone. Endpoint deprecated — Asaas foi substituído por MP.
   Mantido só pra retornar erro claro se algum webhook antigo ainda chegar.
