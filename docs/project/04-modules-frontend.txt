===============================================================================
BOUNTY WORKFLOW — MÓDULOS FRONTEND
===============================================================================
Inclui: src/app (pages + layouts), src/components.
Padrão: Next.js App Router. Server Components por default; "use client"
só quando precisa de state/effects/handlers no navegador.

-------------------------------------------------------------------------------
LAYOUTS E ROUTE GROUPS
-------------------------------------------------------------------------------

 src/app/layout.tsx
   Root layout. Define <html lang="pt-BR">, metadata, og:, robots.
   Wrappa todo o app no ThemeProvider (dark default).

 src/app/error.tsx
   Error boundary global.

 src/app/not-found.tsx
   Página 404 customizada.

 src/app/(public)/layout.tsx
   Layout público: PublicHeader no topo, footer simples com links
   legais e /como-funciona.

 src/app/(auth)/layout.tsx
   Layout mais enxuto — header do logo + fundo suave — usado só
   por /auth/login e /auth/signup.

 src/app/(app)/layout.tsx
   Layout autenticado. Chama requireUser(); se não tiver user,
   redirect pra /auth/login. Renderiza <AppShell>.

 src/app/(app)/admin/layout.tsx
   Sub-layout admin. Chama isAdmin() e bloqueia acesso a não-admins.

COMPONENTES: CAMPANHAS  (src/components/campaigns/*)
-------------------------------------------------------------------------------

 timeline.tsx
   Recebe CampaignWithCounts[] e renderiza uma lista de cards.
   Cada card: título, protocolo, deadline, barra de progresso
   (done_count / step_count), link para /campaigns/[id].

 task-card.tsx  ("use client")
   Card de um Step. Ícone muda pelo status (Circle / Loader2 / CheckCircle2).
   Botão cicla estado: todo -> in_progress -> done -> todo. Usa useTransition
   + toggleStepAction. Opacity 70% quando done.

 workflow-canvas.tsx  ("use client")
   React Flow sobre planning_json da campanha. Nodes = steps, edges = conexões
   opcionais. Drag-and-drop. Persiste via action.

 add-step-form.tsx  ("use client")
   Form inline (title obrigatório, description opcional, due_date) que chama
   addStepAction via useTransition.

 attachment-dropzone.tsx  ("use client")
   Drag-and-drop pra upload no bucket "attachments". Path: {user_id}/{campaign_id}/{file}.
   Lista anexos existentes com download.

-------------------------------------------------------------------------------
STYLING
-------------------------------------------------------------------------------
Tailwind com paleta custom "mint" (50-900) + cores de tema (background,
foreground, muted, card, border, ring, primary, destructive) ligadas a
CSS vars em globals.css. Dark mode por classe.

Componentes usam class-variance-authority (cva) + cn() pra variantes.
